
**arp-scan [опции]**

Инструмент для сканирования локальной сети через ARP и обнаружения активных хостов.<br>Целевые хосты должны быть указаны в командной строке, если не используется опция `--file` или `--localnet`. `arp-scan` использует необработанные, т.н. "сырые" данные, что в некоторых ОС требует повышенных привилегий.<br>Целями сканирования могут быть IPv4-адреса или имена хостов. Можно также использовать CIDR-нотацию, диапазоны и `сеть:маску`.
| Опция | Действие                                                                         |
| :---- | :------------------------------------------------------------------------------- |
|Основные опции   |   |
| `--help` или `-h`|Показывает справочную информацию и завершает выполнение программы |
| `--verbose` или `-v`|Управляет уровнем детализации при выполнении программы. Поддерживает от `-v` до `-vvv` для максимально подробного отчета|
| `--version` или `-V`|Показывает текущую версию программы,а также версию библиотеки `libpcap`|
|  `--interface=<интерфейс>` или `-I`|Использовать выбранный сетевой интерфейс. Если опция не определена то `arp-scan` автоматически выбирает первый сетевой интерфейс со статусом `UP` (интерфейс loopback исключается из поиска)    |
|Выбор хостов   |   |
|`--file=<файл>` или `-f <файл>`|Позволяет читать список целей (IP-адресов или имен хостов) из файла, а не указывать их прямо в командной строке.<br>Файл должен содержать один адрес или шаблон на строку.<br>Специальный символ `-` означает чтение из `stdin`, то есть можно передавать список через конвейер:<br>`cat targets.txt` \| `sudo arp-scan -f -`|
|`--localnet` или `-l`|Автоматически генерирует список IP‑адресов для сканирования на основе конфигурации сетевого интерфейса. При генерации списка также включаются: адрес сети (`.0`) и broadcast‑адрес (`.255`)|
|Файлы соответствия MAC‑адресов и производителей устройств|   |
|`--ouifile=<файл>` или `-O <файл>`|Позволяет указать файл с базой OUI (MAC- производитель), который `arp-scan` будет использовать для сопоставления MAC- адреса и производителя.<br>По умолчанию используется файл `ieee-oui.txt`, который сначала ищется в текущей директории. Если не найден — используется файл по умолчанию: `/usr/share/arp-scan/ieee-oui.txt`   |
|`--macfile=<файл>` или `-m <файл>`|Использует пользовательский файл сопоставления MAC-адресов и производителей. Позволяет добавлять или переопределять производителей устройств, которых нет в стандартной OUI-базе.<br>По умолчанию используется файл `mac-vendor.txt`, который сначала ищется в текущей директории. Если не найден — используется  файл `/etc/arp-scan/mac-vendor.txt`|
|Управление выходными данными   |   |
|`--quiet` или `-q`|Показывает минимальный вывод. Отображаются только IP и MAC -адреса.|
|`--plain` или `-x`|Убирает заголовок и итоговую статистику. Выводит только строки с найденными хостами. Полезно при парсинге.|
|`--ignoredups` или `-g`|Игнорирует дубликаты ответов ARP.|
|`--rtt` или `-D`|Вычисляет RTT (Round Trip Time) — время прохождения пакета.|
|`--format=<значение>` или `-F <значение>`|Позволяет задать собственный формат вывода результатов сканирования. Формат задаётся строкой, где используются переменные вида:<br>`${field}`, разделяемые одинарными кавычками.<br><br>Каждая переменная заменяется соответствующим значением:<br>`${ip}` - IPv4-адрес хоста<br>`${name}` - hostname (если используется `--resolve)`<br>`${mac}` - MAC-адрес<br>`${hdrmac}` - MAC-адрес источника Ethernet<br>`${vendor}` - производитель устройства<br>`${padding}` - padding в ARP-пакете<br>`${framing}` - тип Ethernet-фрейма<br>`${vlan}` - VLAN ID<br>`${proto}` - ARP-протокол<br>`${dup}` - номер дубликата пакета<br>`${rtt}` - RTT (если используется `--rtt`)<br><br>Также можно задать ширину поля:<br>`${field;ширина}`<br><br>Кроме того можно использовать специальные символы escape- последовательностей<br>`\n` - новая строка<br>`\r` - возврат каретки<br>`\t` - табуляция<br>`\` - отключает спецзначение символа|
|Управление порядком сканирования IP-адресов|   |
|`--random` или `-R`|Перемешивает (рандомизирует) список IP-адресов перед сканированием. Используется, чтобы:<br><li>чтобы не создавать последовательную нагрузку на сеть;<li>чтобы сложнее было обнаружить сканирование;<li>для равномерного распределения ARP-запросов|
|`--randomseed=<число>`|Задаёт seed (начальное значение) для генератора случайных чисел. Это нужно, чтобы получать одинаковый случайный порядок адресов при каждом запуске.|
