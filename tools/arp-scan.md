
**arp-scan [опции]**<br>
*arp-scan v1.10.0*

Инструмент для сканирования локальной сети через ARP и обнаружения активных хостов.<br>Целевые хосты должны быть указаны в командной строке, если не используется опция `--file` или `--localnet`. `arp-scan` использует необработанные, т.н. "сырые" данные, что в некоторых ОС требует повышенных привилегий.<br>Целями сканирования могут быть IPv4-адреса или имена хостов. Можно также использовать CIDR-нотацию, диапазоны и `сеть:маску`.
| Опция | Действие                                                                         |
| :---- | :------------------------------------------------------------------------------- |
|Основные опции   |   |
| `--help` или `-h`|Показывает справочную информацию и завершает выполнение программы |
| `--verbose` или `-v`|Управляет уровнем детализации при выполнении программы. Поддерживает от `-v` до `-vvv` для максимально подробного отчета|
| `--version` или `-V`|Показывает текущую версию программы,а также версию библиотеки `libpcap`|
|  `--interface=<интерфейс>` или `-I`|Использовать выбранный сетевой интерфейс. Если опция не определена то `arp-scan` автоматически выбирает первый сетевой интерфейс со статусом `UP` (интерфейс loopback исключается из поиска)    |
|Выбор хостов   |   |
|`--file=<файл>` или `-f <файл>`|Позволяет читать список целей (IP-адресов или имен хостов) из файла, а не указывать их прямо в командной строке.<br>Файл должен содержать один адрес или шаблон на строку.<br>Специальный символ `-` означает чтение из `stdin`, то есть можно передавать список через конвейер:<br>`cat targets.txt` \| `sudo arp-scan -f -`|
|`--localnet` или `-l`|Автоматически генерирует список IP‑адресов для сканирования на основе конфигурации сетевого интерфейса. При генерации списка также включаются: адрес сети (`.0`) и broadcast‑адрес (`.255`)|
|Файлы соответствия MAC‑адресов и производителей устройств|   |
|`--ouifile=<файл>` или `-O <файл>`|Позволяет указать файл с базой OUI (MAC- производитель), который `arp-scan` будет использовать для сопоставления MAC- адреса и производителя.<br>По умолчанию используется файл `ieee-oui.txt`, который сначала ищется в текущей директории. Если не найден — используется файл по умолчанию: `/usr/share/arp-scan/ieee-oui.txt`   |
|`--macfile=<файл>` или `-m <файл>`|Использует пользовательский файл сопоставления MAC-адресов и производителей. Позволяет добавлять или переопределять производителей устройств, которых нет в стандартной OUI-базе.<br>По умолчанию используется файл `mac-vendor.txt`, который сначала ищется в текущей директории. Если не найден — используется  файл `/etc/arp-scan/mac-vendor.txt`|
|Управление выходными данными   |   |
|`--quiet` или `-q`|Показывает минимальный вывод. Отображаются только IP и MAC -адреса.|
|`--plain` или `-x`|Убирает заголовок и итоговую статистику. Выводит только строки с найденными хостами. Полезно при парсинге.|
|`--ignoredups` или `-g`|Игнорирует дубликаты ответов ARP.|
|`--rtt` или `-D`|Вычисляет RTT (Round Trip Time) — время прохождения пакета.|
|`--format=<значение>` или `-F <значение>`|Позволяет задать собственный формат вывода результатов сканирования. Формат задаётся строкой, где используются переменные вида:<br>`${field}`, разделяемые одинарными кавычками.<br><br>Каждая переменная заменяется соответствующим значением:<br>`${ip}` - IPv4-адрес хоста<br>`${name}` - hostname (если используется `--resolve)`<br>`${mac}` - MAC-адрес<br>`${hdrmac}` - MAC-адрес источника Ethernet<br>`${vendor}` - производитель устройства<br>`${padding}` - padding в ARP-пакете<br>`${framing}` - тип Ethernet-фрейма<br>`${vlan}` - VLAN ID<br>`${proto}` - ARP-протокол<br>`${dup}` - номер дубликата пакета<br>`${rtt}` - RTT (если используется `--rtt`)<br><br>Также можно задать ширину поля:<br>`${field;ширина}`<br><br>Кроме того можно использовать специальные символы escape- последовательностей<br>`\n` - новая строка<br>`\r` - возврат каретки<br>`\t` - табуляция<br>`\` - отключает спецзначение символа|
|Управление порядком сканирования IP-адресов|   |
|`--random` или `-R`|Перемешивает (рандомизирует) список IP-адресов перед сканированием. Используется, чтобы:<br><li>чтобы не создавать последовательную нагрузку на сеть;<li>чтобы сложнее было обнаружить сканирование;<li>для равномерного распределения ARP-запросов|
|`--randomseed=<число>`|Задаёт seed (начальное значение) для генератора случайных чисел. Это нужно, чтобы получать одинаковый случайный порядок адресов при каждом запуске.|
|Временные параметры сканирования и повторные попытки отправки пакетов|   |
|`--retry=<число>` или `-r <число>`|Общее количество попыток отправки ARP-запроса каждому хосту.|
|`--backoff=<число_с_точкой>` или `-b <число_с_точкой>`|Коэффициент увеличения тайм-аута между повторными попытками ARP-запроса (backoff). Значение по умолчанию 1.5<br>После каждой попытки сканирования время ожидания ответа увеличивается. Новый timeout вычисляется так:<br>`новый_таймаут = предыдущий_таймаут × backoff`|
|`--timeout=<число>` или `-t <число>`|Начальное время ожидания ответа от хоста в миллисекундах.   |
|`--interval=<значение>` или `-i <значение>`|Минимальный интервал между отправкой ARP-пакетов, ограничивает скорость сканирования.<br>По умолчанию значение указывается в миллисекундах.Можно указать микросекунды, добавив u|
|`--bandwidth=<значение>` или `-B <значение>`|Ограничивает скорость отправки ARP-пакетов по пропускной способности (бит/с).<br>Параметр ограничивает скорость отправки ARP-запросов по объёму трафика, а не по времени между пакетами.<br>Значение указывается в битах в секунду (bps).Можно использовать суффиксы:<br>`K` - `килобиты`<br>`М` - `мегабиты`   |
|Использование DNS для сканирования   |   |
|`--numeric` или `-N`|Отключает использование DNS для целей сканирования. Это значит, что в списке целей можно указывать только IP-адреса, а не доменные имена.   |
|`--resolve` или `-d`|Пытается определить hostname устройства по его IP-адресу. То есть после получения ответа ARP выполняется обратный DNS-запрос (reverse DNS).|
|Управление исходящими ARP-пакетами   |   |
|`--arpsha=<MAC_адрес>` или `-u <MAC_адрес>`|**ARP Source Hardware Address**<br>Устанавливает MAC-адрес отправителя внутри ARP-пакета.<br>Важно: меняет только поле внутри ARP и НЕ меняет MAC-адрес Ethernet-кадра|
|`--arptha=<MAC_адрес>` или `-w <MAC_адрес>`|**ARP Target Hardware Address**<br>Задает MAC адрес получателя в ARP-данных.<br>`ar$tha (Target Hardware Address)`<br>По умолчанию: `00:00:00:00:00:00`, потому что в ARP request MAC цели неизвестен.|
|`--arphrd=<число>` или `-H <число>`|**ARP Hardware Type**<br>16-битное значение, определяющее тип канального уровня.<br>По умолчанию: `1 = Ethernet (ARPHRD_ETHER)`<br>Распространённые значения:<li>1 - Ethernet<li>6 - IEEE 802<li>15 - Frame Relay<li>19 - ATM<br>   |
|`--arppro=<тип_протокола>` или `-p <тип_протокола>`|**Protocol Type**<br>Тип протокола, для которого выполняется ARP. По умолчанию: `0x0800` - IPv4<br>Распространённые значения:<li>0x0800 - IPv4<li>0x86DD - IPv6<li>0x0806 - ARP|
|`--arphln=<число>` или `-a <число>`|**Hardware Address Length**<br>Размер адреса оборудования (MAC).<br>По умолчанию: `6` - потому что MAC- адрес равен 6 байтам.<br>Важно: эта опция меняет только значение поля, но не длину самих адресов.   |
|`--arppln=<число>` или `-P <число>`|**Protocol Address Length**<br>Длина адреса сетевого уровня.<br>По умолчанию: `4` - потому что IPv4 равен 4 байтам<br>   |
|`--arpop=<число>` или `-o <число>`|**ARP Operation**<br>Тип операции ARP.<br>По умолчанию: `1` - ARP Request<br>Распространённые значения:<li>1 - ARP Request<li>2 - ARP Reply<li>3 - RARP Request<li>4 - RARP Reply|
|`--arpspa=<IP_адрес>` или `-s <IP_адрес>`|**Source Protocol Address**<br>IP адрес отправителя<br>По умолчанию устанавливается IP- адрес интерфейса отправителя<br>Можно указать **специальное значение**: `dest` - устанавливает адресом назначения адрес отправителя.|
|Управление исходящими Ethernet- кадрами|   |
|`--srcaddr=<MAC_адрес>` или `-S <MAC_адрес>`|**Source Ethernet Address**<br>Устанавливает MAC-адрес источника в заголовке Ethernet. По умолчанию установлен MAC- адрес интерфейса отправителя.<br>Важно: ARP- поле `ar$sha` не меняется.|
|`--destaddr=<MAC_адрес>` или `-T <MAC_адрес>`|**Destination Ethernet Address**<br>Устанавливает MAC-адрес назначения в заголовке Ethernet. По умолчанию установлен широковещательный адрес `FF:FF:FF:FF:FF:FF`<br>Можно установить префиксы unicast или multicast:<li>**Unicast**: первый бит первого байта = 0, то есть первый байт чётный<li>**Multicast**: первый бит первого байта = 1, то есть первый байт нечётный   |
|`--prototype=<тип_протокола>` или `-y <тип_протокола>`|**Ethernet Protocol Type (EtherType)**<br>По умолчанию: `0x0806` - ARP|
|`--llc` или `-L`|Использовать инкапсуляцию LLC/SNAP. Эта опция включает формат IEEE 802.2 + SNAP вместо обычного Ethernet II. Описан в RFC 1042.|
|`--vlan=<число>` или `-Q <число>`|**VLAN- теггинг (802.1Q)**<br>Добавляет VLAN- тег в Ethernet- кадр.<br>Диапазон VLAN ID: `1-4094`   |
|Прочие опции   |   |
|`--limit=<число>` или `-M <число>`|Ограничение количества найденных хостов. Завершает работу после получения `N` ответов.   |
|`--pcapsavefile=<файл>` или `-W <файл>`|Сохраняет все полученные ARP- ответы в файл формата PCAP.   |
|`--snap=<число>` или `-n <число>`|Определяет сколько байт каждого пакета сохранять.<br>По умолчанию: `64` - столько хватит для захвата всей нужной ARP-  информации из заголовка Ethernet- кадра|
|`--retry-send=<число>` или `-Y <число>`|Определяет количество отправок ARP- запросов<br>По умолчанию: `20`|
|`--retry-send-interval=<число>` или `-E <число>`|Интервал между ARP запросами<br>По умолчанию: `20 ms`<br>Можно указать микросекунды, добавив u    |
|`--padding=<hex>` или `-A <hex>`|Позволяет добавить дополнительные байты после ARP- данных.   |
